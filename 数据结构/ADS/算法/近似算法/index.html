近似算法

近似算法（Approximation） #

概述 #

用于解决困难问题，如NPC问题。

对于NPC问题的处理方法：

  • N很小，$O(2^N)$也可以接受。

  • 总体问题很复杂，但可以在多项式时间解决一些重要的特殊情况。

  • 寻找一个能在多项式时间得出的次优解（near-optimal solution）。→ 渐近算法

近似比例（approximation radio） #

定义 #

若对于任意大小的输入$n$，渐近算法求解的开销$C$不超过最优解开销$C*$的$\rho (n)$因子，则该算法的渐近比例为$\rho(n)$。称为$\rho(n)$渐近算法。

近似方案（approximation scheme） #

一个最优解问题的渐近方案是一个输入为问题instance和一个正数$\epsilon$的渐近算法，使其对于任何确定的$\epsilon$，都是一个$(1+\epsilon)$渐近算法。

若对于任意确定的$\epsilon >0$，渐近方案能在多项式时间内得出结果，则称为PTAS（polynomial-time approximation scheme）。

通常PTAS的时间复杂度形如$O(n^{2/\epsilon})$。随着$\epsilon$减小，时间复杂度增大。

有一种特殊情况复杂度形如$O((1/\epsilon)^2n^3)$，优于上面的情况，这种PTAS称为FPTAS（fully polynomial-time）。

fully表示时间复杂度中的两个变量$n$和$\epsilon$都是多项式形式。

实例1：装箱问题 #

Bin Packing，NP-Hard问题。

给出大小分别为$S_1,S_2,……S_N$的$N$个物品，每个$S_i$都有$0<S_i\le 1$。每个背包有单位空间，将这些物品装入数量最少的背包里。

decision version： 给出K个背包和N个物品的大小，求解是否能装下。

NP-Compelete问题。

Next Fit #

依次遍历每个物品，若上一个背包能放下则放入上一个背包，否则新开一个背包。线性时间复杂度。

定理：

若$M$为物品集$I$的最优解（即最少背包数），则Next Fit近似算法的结果一定不超过$2M-1$，且存在解法使得结果刚好等于$2M-1$。

证明：

反证法：即证明若NF算法结果为$2M$或$2M+1$，则最优解的结果一定大于等于$M+1$。

近似比例为2，需要更好的方案。

First Fit #

搜索第一个满足能装下该物品的背包，若存在，则放入该背包中；若不存在，则新开一个背包。时间复杂度为$O(NlogN)$。

定理：

若$M$为物品集$I$的最优解（即最少背包数），则First Fit近似算法的结果一定不超过$17M/10$，且存在解法使得结果刚好等于$17(M-1)/10$。

近似比例为1.7。

Best Fit #

将物品放置在“最拥挤”的背包里，即放下物品后剩余空间最小。时间复杂度为$O(NlogN)$。结果同样$\le 1.7M$。

Worst Fit #

将物品放置在“最空闲”的背包里，即放下物品后剩余空间最大。

对比 #

另一个例子：

最优解为按列三个一组，结果为6；而按顺序放入背包得出的结果为10。

On-line算法 #

按输入顺序依次计算，进行下一步后无法撤回上一步的选择。

由于无法预知输入什么时候截止，因此on-line算法无法一直给出最优解。

定理：

存在一种输入使得背包问题的任何on-line算法得出的结果至少为最优结果的$5/3$。（趋近于1.7）

如BF、WF、AWF、AF等算法的近似比例均约为1.7。

Off-line算法 #

先获取所有输入列表，再进行计算。

问题所在：较大的物品

解决办法：将输入的物品大小按降序排列，再按BF或FF算法计算。称为BF/FF decreasing(primary)算法。

定理：

若$M$为物品集$I$的最优解（即最少背包数），则FF decreasing算法的结果一定不超过$11M/9+6/9$，且存在解法使得结果刚好等于$11M/9+6/9$。

近似比例约为1.2。

实例2：背包问题 #

商店的货架上摆放着不同重量和价值的商品，一个小偷在商店行窃，他携带的背包只能装固定重量的商品。装哪些商品才能获得最大的收益呢？在限定条件内找到最佳的物品组合，这样的问题统称为背包问题（Knapsack Problem）。

分类 #

  • 部分背包问题： 所有物品是可再分的，即允许将某件物品的一部分（例如 1/3）放入背包；

  • 0-1 背包问题： 所有物品不可再分，要么整个装入背包，要么放弃，不允许出现“仅选择物品的 1/3 装入背包”的情况；

  • 完全背包问题： 不对每一件物品的数量做限制，同一件物品可以选择多个装入背包；

  • **多重背包问题： **每件物品的数量是有严格规定的，比如物品 A 有 2 件，物品 B 有 3 件。

部分背包问题（fractional version） #

背包容量为$M$，给出$N$个物品，每个物品$i$有重量$w_i$和收益$p_i$。若$x_i$为物品$i$放入背包的比例，则收益为$p_ix_i$。

最优解定义为：解出$x_i$，使得$\Sigma_{i=1}^{n}{p_ix_i}$取得最大值。限定条件为：$\Sigma_{i=1}^{n}{w_ix_i}\le M$，且$x_i \in [0,1]$。

可以用简单的贪婪算法解决：

  • 将一个物品放入背包作为一个步骤。
  • 贪婪算法理论：①收益最大 ②重量最小 ③收益密度$p_i/w_i$最大（√）。

0-1背包问题 #

$x_i$必须是1或0。即一个物品只能选择全部放入或不放入。NP-Hard问题。

贪婪算法

若应用贪婪算法取收益最大和收益密度最大中的较大值作为结果，其近似比例为2。

证明：

动态规划

定义$W_{i,p}$表示在物品集合${1,……,i}$中，总收益为$p$时的最小重量。

  • 若装入物品$i$：$W_{i,p}=w_i+W_{i-1,p-p_i}$。
  • 若放弃物品$i$：$W_{i,p}=W_{i-1,p}$。
  • 若无法达到$p$：$W_{i,p}=\infin$。

时间复杂度：考虑最坏情况，$i=1,……,n,\\ p=1,……,np_{max}$，此时时间复杂度为$O(n^2p_{max})$。

注：该时间复杂度不代表该算法为P问题，由于$p_{max}$不是常数。

当$p_{max}$非常大时，其耗时甚至可能大于穷举法。解决办法：把$profit$进行约化，同除一个数以减小数据大小。

实例3：K中心问题 #

在居民区附近建超市。site(primary)表示居民建筑，center(primary)表示超市中心。

输入：$n$个居民建筑：$s_1,……s_n$。

问题：给定整数$K$，选择$K$个超市中心（集合$C$），使得一个居民建筑到最近的超市中心的最大距离（distance）最小化。

定义distance(primary)：

  • identity：$dist(x,x)=0$
  • symmetry：$dist(x,y)=dist(y,x)$
  • triangle inequality：$dist(x,y)\le dist(x,z)+dist(z,y)$

$dist(s_i,C)=\min_{c\in C}{dist(s_i,c)}=distance$ $from$ $s_i$ $to$ $the$ $cloest$ $center$

$r(C)=\max_{i}{dist(s_i,C)}=smallest$ $covering$ $radius$

目标：找到满足$|C|=K$的超市中心集合$C$，使得$r(C)$最小。

候选的超市中心有无数个。

贪婪算法 #

将第一个中心放在对于一个中心点来说最好的位置上，再一次添加中心点，每次尽可能减小覆盖半径（covering radius）。

存在问题：

改进：Greedy-2r算法

寻找$r(C*)$的上界，$C*$表示最优解。

易知若最小的覆盖半径为$r(C*)$，则贪婪算法的最大覆盖半径不超过$r=2r(C*)$。

理论： 若该算法选择出了大于$K$个中心点，则对于不超过$K$的任意集合$C*$，覆盖半径均有$C*\gt r $。（反证法）

问题： 如何获取$r(C*)$，即覆盖半径的上界？

使用对$r$的二叉搜索。选择一个很大的数$r_{max}$，有$0\lt r \le r_{max}$，每次猜测$r=(0+r_{max})/2$。

①Yes：说明能够找到K个中心点，即最优解覆盖半径小于等于r，此时可以减小r的值。

②No：r的值过小，可以增大。

不断缩小r的范围，直到确定值。近似比例为2。

时间复杂度： 多项式时间。

更聪明的算法：be far away

**理论：该算法将返回拥有$K$个中心点的集合$C$使得$r(C)\le 2r(C)$，其中$C$是最优解，近似比例为2。（反证法）

进一步改进：是否存在近似比例为3/2或4/3的算法？

理论： 除非$P=NP$，否则对于中心点选择问题，不存在$\rho$-approximation算法使得$\rho \lt 2$。（反证法）

证明：如果能在多项式时间内实现$\rho \lt 2$的近似算法，则可以在多项式时间内解决DOMINATING-SET(primary)问题（NPC）。

Dominating set问题有大小为$K$的解当且仅当存在$K$个中心点集合使得$r(C*)=1$。

总结 #

三方面需要考虑：

  • A. 最优性： 结果的最优程度。
  • B. 效率： 计算的开销
  • C. 普适性： 不仅对特殊情况适用。

A+C： 对所有情况的精准解法。如暴力穷举。

A+B： 对特殊情况的精准快速解法。如动态规划。

B+C： 效率+普适性替代结果的质量。如近似算法。

即使$P=NP$，也不能保证同时实现$A+B+C$。
