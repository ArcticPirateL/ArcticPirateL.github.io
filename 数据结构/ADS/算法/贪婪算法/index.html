贪婪算法

贪婪算法 #

概述 #

在Dijkstra算法求解最短路径、Kruskal算法求解最小生成树中都使用了贪婪算法的思想。

用于求解最优问题。通常已知一些约束和最优函数，满足约束的解决方案成为可行方案（feasible solution）。

最优函数值最大的可行方案称为最优方案。

核心思想 #

每个步骤都选择最优解，上一步选择的结果在后续步骤中不会改变，即不能撤销前面步骤，同时每个选择都应确保方案可行。

注意：

  • 贪婪算法只在当前最优解即为总体最优解时能正确解出最优解，即local optimum = global optimum的gool luck情况。
  • 贪婪算法不能保证结果一定为最优，但它给出的结果一定与最优结果取值相近，即启发式算法（heuristics）。因此当寻找实际的最优解需要耗费大量时间时，直觉上可以采用贪婪算法。good enough。

活动选择问题 #

给出需要共用同一资源的一系列时间，记为$S={a_1,a_2,……,a_n}$。每个事件$a_i$发生的时间间隔为$[s_i,f_i)$。

对于事件$a_i$和$a_j$，若有$s_i\ge f_j$或$s_j\ge f_j$，则称为两个事件兼容，即两个事件的时间不重合。

假设：$f_1\le f_2\le……\le f_{n-1}\le f_n$。

目标是选择能够兼容发生的事件的最大子集。

例：

最多能兼容4个事件，为最优解，且最优结果不唯一。

动态规划求解 #

定义$S_{ij}$为起始时间在$a_i$开始之后且结束时间在$a_j$结束之前的解。

将$S_{ij}$分为两个子问题，进行divide-conquer。

时间复杂度为$O(N^3)$。

贪婪算法求解 #

不同的贪婪规则：

  • 挑选在不与前面重合的前提下，最早开始的事件。

  • 挑选在不与前面重合的前提下，时间最短的事件。

  • 挑选在不与前面重合的前提下，与其余事件冲突最少的事件。

  • 挑选在不与前面重合的前提下，最早结束的事件。即尽快使资源得到释放。

正确性证明 #

需要证明：①算法的结果满足时间不重合。 ②结果最优。

定理

对于任意非空的子问题$S_k$，若$a_m$是$S_k$中最早结束的事件，则$a_m$一定是$S_k$最优解的集合中的元素。（最优解不一定唯一）

证明：

设$A_k$为某一最优解集合（不一定包括$a_m$），设$a_{ef}$为某个最优解集合中结束时间最早的一个事件。

①若$a_m$与$a_{ef}$是同一个事件，则无需额外证明。

②否则，用$a_m$替换$a_{ef}$，得到新的集合$A_k’$。由于$a_m$是整个集合中最早结束的事件，因此有$f_m\le f_{ef}$。

显然$a_m$与这一解集合中的其他事件都不冲突，因此$A_k’$是一个新的最优解集合。

最终解决办法

每次选择最先结束的事件，尾递归可以转换为迭代求解，因此总的时间复杂度为：$O(NlogN)$。

问题：能否将“最先结束”这一条件替换为“最迟开始”？

答：可以，能够通过证明。

另一种DP求解 #

其中$c_{1,j}$表示从第一个事件到第j个事件间的最优解，$k(j)$表示在$a_j$开始前结束且距离$a_j$最近的事件。

相较于第一种动态规划，在证明了上述定理的基础上，把i、j双重循环简化为单层循环。

$c_{1,j-1}$表示不选取$a_j$时的最优解，$c_{1,k(j)}$表示在$a_j$之前与$a_j$不冲突的最优解，$c_{1,k(j)}+1$表示选取$a_j$时的最优解。

如果修改问题为：每个事件带有一定权重，而最优解指的是所有事件权重之和最大的解

此时动态规划算法依然是正确的，但贪婪算法不一定正确。

贪婪策略的要素 #

  • 将最优问题分步骤解决，使得每步中需要做出一个选择。
  • 需要证明，总存在至少一个最优解包含贪婪算法的选择（与上述正确性证明类似），这保证了贪婪算法是安全可行的。
  • 需要证明，存在最优子结构。即用贪婪算法选择出的最优子解+剩余部分的最优子解=原问题的最优解。

霍夫曼编码问题 #

问题描述 #

霍夫曼编码通常用于文件压缩。

如一个文件由一个长度为1000的字符串组成，其中只包括a、u、x、z四个字符。正常情况下需要8000 bits来存储这个字符串。

但由于只包含四个字符，只用2 bits即可区分不同的字符，此时共需要2000 bits来存储这个字符串。

因此，当字符串中存在C中不同的字符时，可以用$logC$个bits来表示每个字符。

例如字符串：aaaxuaxz

为使存储空间更加节省，我们需要获取一些额外的信息，包括：

  • frequency：某个符号出现的次数（频率）。

    此时可以使用可变长度的bits对某个符号进行编码。如a=0, u=110, x=10, z=111。

    最终编码为：00010110010111

    可见其长度明显小于使用1B或2 bits的编码，实现了文件压缩效果。

    注：当所有字符出现的频率都相同时，这种方法不会有压缩的效果。

问题：如何得到这些字符的编码？

使用一个二叉树（trie）表示原始的编码。

定义：对于一个字符$C_i$，$d_i$为其在二叉树中的深度，$f_i$为其在文件中出现的频率。定义这种编码的 $cost=\sum{d_if_i}$。

​ 显然对于一个字符，深度表示其编码长度，频率表示其出现次数，因此$d_if_i$表示其在整个文件中总共的编码长度。

​ 因此$d_if_i$的累加和表示这种编码方式编码的文件总长度。

最优解：即cost最小的编码方式。frequency越大的字符，取的depth越小。

存在的问题：如果已知每个字符的编码，求原字符串。如果将u的编码从110替换为100，则逆向的结果不唯一！

因此要求：任何一个字符的编码不能是另一个字符的前缀。即当u=100时，x=10是u的前缀，此时结果不唯一。

在二叉树的结构上体现为：每条路径上不能有大于等于2个字符节点，即所有字符节点必须是叶子节点。

​ 同时由于要达到最小的cost，因此必须为满二叉树（每个节点要么是叶子节点，要么有两个儿子）。

原理：如果存在degree为1的节点，则可以删除该节点，得到更短的编码长度，不符合cost最小的最优结果。

这种编码称为前缀编码（prefix code）。

一个结论：包含$n$个字符的字母表$C$，其中所有字符最终的最优前缀编码长度一定小于等于$n-1$。

**目标：**找到能使所有字符位于叶子节点，且总cost最小的满二叉树。

贪婪算法求解 #

首先明确，求解这一满二叉树需要按从底层到顶层的顺序构建二叉树。

要求拥有最小频率的字符编码长度最长，即到根节点的路径最长。

算法步骤：

  • 定义节点的weight为其儿子的weight之和，单个节点的初始weight为其frequency的值。

  • 使用最小堆结构，初始化为将所有的单个字符节点插入到堆中。

  • 每次循环定义一个新节点，删除两次堆中的最小节点，分别attach到新节点的左右儿子。

  • 更新新节点的权重，并将新节点插入到最小堆中，继续循环。

这种算法保证了depth与frequency乘积的总和达到最小。

最后需要在每个分支添上0或1，表示最终编码。

时间复杂度为：$T=O(ClogC)$，其中$C$表示字符个数。

正确性证明 #

  • 证明最优解包含贪婪算法解出的结果。

    引理1：设$C$为字母表，$x,y$是其中frequency最小的两个字符。

    ​ 则一定存在一个$C$的最优前缀编码，其中$x$和$y$有着相同的depth，两者的编码差别只在最后一位。

    需要证明：已知$T$为一个最优解，对字符重新排序使得$x$和$y$有相同的depth，证明$cost(T’)\le cost(T)$，则$T’$也是最优解。

  • 证明存在最优子结构。

    引理2：已知字母表$C’$中字符$z$的frequency等于字母表$C$中$x$和$y$的frequency之和。

    ​ 则如果将$C’$中的$z$替换为一个左右儿子各为$x$和$y$的新节点，新的$T$也是$C$的最优解。

    本质：证明其余问题的最优解（$T’$）+ 子问题的最优解（$x,y$同深度）= 原始问题的最优解（$T$）

分治算法vs贪婪算法vs动态规划 #

对于每个贪婪算法，总存在一个较为复杂的动态规划算法能解决相同的问题。

分治算法 #

优点：

  • 分治算法的思想简单，易于理解。
  • 对于某些问题，分治算法可以有效地减少计算量。
  • 分治算法可以很容易地并行化，适用于多核计算。

缺点：

  • 分治算法中，递归会导致函数调用次数的增加，因此在处理大规模数据时，可能会导致栈溢出。
  • 分治算法不适用于某些问题，例如求最长公共子序列等问题。

适用条件：

  • 问题可以划分为多个相互独立的子问题。
  • 子问题的规模比原问题小，并且解决子问题的时间复杂度较小。
  • 子问题的解可以合并为原问题的解。

贪婪算法 #

优点：

  • 简单：贪心算法通常比较简单，易于实现和理解。
  • 高效：贪心算法的时间复杂度通常比较低，能够在较短的时间内得到一个可行的解。
  • 可以解决一些特殊问题：贪心算法可以很好地解决一些特殊的最优化问题，如霍夫曼编码、活动选择问题等。

缺点：

  • 局部最优解：贪心算法只考虑了当前状态下的最优解，无法保证一定能够得到全局最优解。
  • 难以设计：有些问题难以用贪心算法求解，需要深入的分析和设计。

适用条件：

  • 子问题的最优解能够递推到整个问题的最优解。
  • 贪心策略能够保证问题的最优解。

动态规划算法 #

优点：

  • 可以得到全局最优解：动态规划算法可以保证得到全局最优解，比贪心算法更为强大。
  • 可以解决一些无法用贪心算法解决的问题：动态规划算法可以很好地解决一些无法用贪心算法解决的问题，如背包问题、最长公共子序列等。
  • 动态规划算法可以优化一些问题的计算时间，减少不必要的重复计算。
  • 动态规划算法可以处理一些具有多阶段决策过程的问题。

缺点：

  • 相对较慢：动态规划算法需要逐步求解问题的最优解，相对较慢。
  • 需要较大的空间：动态规划算法需要存储中间结果，需要较大的空间。

适用条件：

  • 问题具有最优子结构，即问题的最优解可以由子问题的最优解递推得到。
  • 问题的重叠子问题性质，即子问题之间存在重复计算的情况。
  • 问题可以分解为多个阶段，并且每个阶段可以做出一个决策，影响下一阶段的状态。

总结 #

分治算法 动态规划 贪婪算法
适用类型 通用问题 优化问题 优化问题
子问题结构 每个子问题不同 很多子问题重复（不独立） 只有一个子问题
最优子结构 不需要 必须满足 必须满足
子问题数 全部子问题都要解决 全部子问题都要解决 只要解决一个子问题
子问题在最优解里 全部 部分 部分
选择与求解次序 先选择后解决子问题 先解决子问题后选择 先选择后解决子问题
