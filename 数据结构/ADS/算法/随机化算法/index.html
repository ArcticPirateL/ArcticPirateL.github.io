随机化算法

随机化算法（Randomized Alogorithms） #

随机化的定义 #

  • the world behaves randomly：随机生成乱序的输入并使用传统算法解决 $\Rightarrow$ 平均情况分析（Average-case Analysis）。
  • the algorithm behaves randomly：当输入情况最坏时算法采用随机决策的方式求解。

两种随机化目标：

  • efficient algorithm： yield the correct answer with high probability.注重效率高。
  • deterministic algorithm： always correct and run efficiently in expectation.注重正确性。

概率前置知识 #

  • $Pr[A]$：事件$A$发生的概率。

  • $\overline A$：事件$A$的补集，即不发生。

  • $E[X]$：随机变量$X$的期望（平均值）。

    $E[X]=\Sigma_{j=0}^{\infin}j\cdot Pr[X=j]$。

实例1：Hiring问题 #

  • 雇佣一个助手，共$N$天，每天面试一个人。

  • 面试开销：$C_i « $雇佣开销：$C_h$。

分析开销，假设$N$天面试雇佣了$M$个应聘者，则总开销为：$O(NC_i+MC_h)$。目标为使总开销最小化。

原始解法 #

从0开始，每次遇到更好的应聘者，解雇此前最好的，雇佣当前的。

最坏情况：候选人的质量按升序排列。此时开销约为$O(NC_h)$。

最好情况：候选人的质量按降序排列。

改进方向：

  • 提高解决最坏情况的效率。
  • 尝试避免出现最坏情况。

若候选人随机顺序出现。设$X$为雇佣次数，找到$X$的期望。核心问题是如何知道$Pr[X=j]$。

假设：每个人称为当前最佳的可能性相等。

结论： 若候选人随机出现，则开销为$O(C_hlnN+NC_i)$，明显小于最坏情况。因此需要随机化算法。

随机化算法 #

无论候选人以什么顺序出现，都进行一次随机化。避免了最坏情况，但增大了开销。

Randomized Permutation Algorithm的核心难题是如何实现equally likely。

目标：重新排列给定的数组$A[]$。

方法：为每个$A[i]$分配一个随机的优先级$P[i]$，并按优先级排序。

断言：PermuteBySorting能够生成一个uniform的随机排列，若所有优先级互不相同。

uniform的含义为所有$N!$中随机排列的情况出现的可能性相等。

Online Hiring算法 #

只雇佣一次。通过第1-k个候选者选出bestQ，但不更新best，也不进行录用。第k+1-n个候选者与当前的bestQ比较，一旦大于就录用。

显然，不一定选出真正最优的候选者。

问题：

  • 对于给定的k，我们成功选择出最优候选者的概率是多少。
  • 选择k为多少能够使这个概率最大化。

问题1：给定k，选择正确的概率

定义情况$S_i$表示选择出最优候选者，且最优的候选者出现在第$i$个位置。

因此最终选出最优候选者的概率为$i$出现在第$k$个位置之后的概率之和。

因此$S_i$发生的条件为：①$A$：最优候选者的位置为$i$。②$B$：第$k+1\sim i-1$的候选者均未被录用。

由于两者相互独立，因此$Pr[S_i]=Pr[A]\cdot Pr[B]$。

问题2：求k，使概率最大

即找到对应的$k$使得左侧最大化。

最终解出当$k=\frac{N}{e}$时，我们选择出最优候选者的概率至少为$\frac1e$。(e=2.71828……)

即使用总人数的前三分之一作为标杆，寻找后三分之二中第一个超过标杆的人，选择成功的概率至少为三分之一。

实例2：Quicksort问题 #

快速排序平均情况的时间复杂度为$O(NlogN)$，最坏情况为$O(N^2)$。我们称这种版本为deterministic quicksort。

一种随机化办法：每次随机选择pivot的值。

  • 定义Central splitter表示能够保证每边至少包含$n/4$的pivot点。

  • 定义Modified Quicksort表示每次递归之前都选择Central splitter的快速排序算法。

**断言：**我们寻找Central splitter的迭代次数最多为2。

每次随机化命中Central splitter空间的概率为$1/2$。

定义type：定义了第j次递归中Central splitter的最大空间。

断言： type j最多有$(\frac43)^{j+1}$个子问题。

总结 #

offline hiring algorithm：always correct but less efficient.

online hiring algorithm：efficient but no longer guarantee the correctness.

modified quicksort：always efficient and correctness but NlogN is an expectation not the real case.
