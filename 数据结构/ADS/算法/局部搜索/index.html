局部搜索

局部搜索（Local Search） #

同样是使用近似方法解决问题。特点是关注局部最优解。

初始猜测 → 搜索局部最优解 → 用局部最优解替换上一步猜测 → 循环。

需要考虑的要素：①初始点的选择 ②步长的选择

是否能在有限步数内终止程序？

概念 #

Local #

  • 定义neighborhoods为当前点周围一个可以达到的集合。
  • 局部最优解为neighborhoods集合中的最优选择。

Search #

  • 从一个可行解开始，找到其neighborhoods中的最优解。
  • 若没有更优则停止程序。

邻域关系 #

  • $S\sim S’$：$S’$是$S$的邻域内的解（neighboring solution）。即$S$可以通过小范围修改得到$S’$。

  • $N(S)$表示$S$的邻域，即集合${S’:S\sim S’}$。

Gradient descent：梯度下降法。

实例1：Vertex Cover问题 #

Decision Version：

给出无向图$G=(V,E)$和整数$K$，是否存在节点集合$V’\subseteq V$使得$|V’|$最大为$K$，且$G$中的每条边至少有至少有一个节点属于$V’$？

Optimazition Version：

给出无向图$G=(V,E)$，找到一个最小集合$S\subseteq V$使得$G$中的每条边至少有至少有一个节点属于$S$？

要素：

  • 可行解集合$FS$：所有满足vertex covers的解集合。显然$V\in FS$。

  • 开销：$cost(S)=|S|$。

  • 局部：$S’$可以由$S$加或减一个节点得到。由于要使开销最小，因此选择“减”一个节点。

    每个$S$最多有$|V|$个邻域（删除每个节点）。

  • 搜索：从$S=V$开始，删除一个节点，并判断$S’$是否是开销更小的vertex cover。

Case 0：只有节点没有边 #

只有一个全局最优解。此时S为空集。

Case 1：中心节点cover所有边 #

显然全局最优解为只有中心节点的集合。但若从所有节点开始，第一次就删除了中心节点，则无法删除剩余任何一个节点。

Case 2：节点直线排列没有支路 #

显然中间三个节点为开销最少的vertex cover。但若第一步删除了这三个节点之一，无法删除其他两个节点。

存在的问题： 无法撤回上一步操作，很容易陷入局部最优后终止程序。

改进：Metropolis算法。

Metropolis算法 #

将“删除一个节点”修改为随机选择邻域内所有解，即可以增加节点。

若当前操作比当前最优操作开销更大，仍需要选择是否更新当前解。

如case 1，仍存在可能将中心节点加回来。但对于case 0，可能导致在添加或删除中无限循环。

因此metropolis算法的效果取决于$probability$的定义，即取决于T。

实际上T与温度有关，Metropolis是一个物理学家，这个算法有物理学背景，T指温度。

由于很难确定合适的T，因此改进为模拟退火法。

模拟退火（Simulated Annealing） #

模拟退火：一种用于求解优化问题的随机搜索算法，通过模拟固体在冷却过程中的能量最小化来寻找问题的全局最优解。

思想：材料从高温开始逐渐冷却，使其有足够的时间在一系列较低的温度下达到平衡。

因此需要一个温度序列。Cooling schedule：$T={T_1,T_2,\ldots}$。

问题 #

对于一个$Hopfield$网络，是否一定存在一个稳定构造，若是，如何找到这个构造？

State_flipping算法 #

从一个随机构造开始（如所有结点为同一个状态），循环检测$S$是否稳定。

若否，选出一个unsatisfied的节点，并将其状态反转，直到检测到$S$为稳定构造。

通过例子可以观察到，一个节点的状态极有可能多次翻转。那么这个程序是否一定能结束呢？（是）

断言：state_flipping算法可以在最多$W=\Sigma_e|w_e|$次（即所有权重绝对值之和）迭代后找到一种稳定构造。

证明：

  • 定义函数$\Phi(S)=\Sigma_{e\\ is\\ good }|w_e|$，即所有good的边的权重绝对值之和。用于衡量how good。

  • 当节点$u$的状态翻转时，与$u$相连的所有边，good → bad && bad → good，其他边不变。

  • 每次翻转后都能保证$\Phi(S)$至少增加1，显然$0\le \Phi(S)\le W$，因此最坏的情况是从0开始，每次只能+1，总迭代次数为$W$。

与局部搜索联系 #

  • 问题： 使$\Phi$最大化。

  • 可行解： 所有的构造集合。

  • $S\sim S’$：$S’$可以由$S$经过一次翻转得到。

断言：state-flipping算法中任何使$\Phi$最大化的局部最优解都能达成稳定构造。

State-flipping算法的时间复杂度最坏情况为$W$，但由于$W$的增长速率无法确定，因此不是多项式时间算法。

是否存在多项式算法仍然是一个开放问题。

实例3：Maximum Cut问题 #

定义 #

给定一个无向图$G=(V,E)$，边的权重均为正整数，找到一种节点分割方式$(A,B)$（如将所有节点分成两个颜色），使得跨越这个分割的所有边的权重之和最大化。即$w(A,B)$最大化。

实际应用：

给出$n$个活动和$m$个人，每个人希望参加其中的两个活动（即每个人作为一条边，把两个节点连接起来）。

将每个活动设置为上午/下午，使得最多数量的人能满足需求（尽可能希望一条边的两个节点颜色不同）。

与局部搜索联系 #

  • 问题： 使$w(A,B)$最大化。
  • 可行解： 任意的分割方式$(A,B)$。
  • $S\sim S’：$$S$可以通过将一个节点从一方移到另一方来得到$S’$。（称为Single-flip neighborhood）

显然，Maximum Cut问题实际上是Hopfield Neural Network问题的特殊情况，即所有边的权重均为正。

  • 不是多项式时间算法。

1. 局部最优解的质量评估 #

断言： 若$(A,B)$定义为局部最优解，$(A*,B*)$定义为全局最优解，则有$w(A,B)\ge \frac{1}{2}w(A*,B*)$。即近似比例约为2。

2. 多项式算法问题 #

一种解决办法：当程序运行到解为big enough时，结束程序。

Big-imporvement-flip算法

只选择能提升$w(A,B)$大小至少为$\frac{2\epsilon}{|V|}w(A,B)$的节点进行翻转。当$\epsilon=0$时，即为State-flipping算法。

断言1：big-improvement-flip算法结束时，返回的$w(A,B)$满足：$(2+\epsilon)w(A,B)\ge w(A*,B*)$。

断言2：big-improvement-flip算法迭代次数最多为：$O(n/\epsilon logW)$，显然为多项式算法。

3. 定义一个更好的local #

  • 邻域的定义应当足够大，使得我们不容易陷入bad local optimal。

  • 领域的定义不能过于大，使得找到局部最优的开销过大。

一种解决办法：

$Single-flip\rightarrow k-flips$，即每次翻转k个节点。此时局部搜索的时间复杂度为$\Theta(n^k)$。

一种启发式算法：

$K-L$ $heuristic$。
