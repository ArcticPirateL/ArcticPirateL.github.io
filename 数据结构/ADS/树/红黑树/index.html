红黑树

红黑树 #

概述 #

AVL树是首个自平衡树，定义了“平衡”的衡量标准（BF）。

Splay树是AVL树的简化版，无需考虑平衡，表现与AVL树相同。

现实应用中存在其他定义“平衡”的方法：如红黑树、B+树，红黑树应用于C++STL中的map等类型。

目标 #

平衡二叉搜索树。

NULL节点也称为NIL。

定义 #

五个特性 #

  • 每个节点是红色或黑色中的一个。
  • 根节点为黑色。
  • 每个NULL叶子节点为黑色。
  • 红色节点的两个儿子一定为黑色。
  • 对于每个节点，其到叶子节点的所有简单路径上黑色节点的个数相同。

black_height：从x到叶子节点的黑色节点的个数，不包括root包括NULL。bh(Tree) = bh(root)

internal_nodes：除NIL节点外的结点。

external_nodes：NIL节点。

定理 #

拥有N个internal_nodes的红黑树，其高度最大为2ln(N+1)。

即使没有AVL更为严格的平衡定义，红黑树也将其高度限制在了O(lgN)的水平。

  • 对于任何节点x，以x为根的所有子树的internal_nodes个数大于等于$2^{bh(x)}-1$。通过数学归纳法证明。

    将x替换为root，则sizeof(x) = N，得到$N>=2^{bh}-1$。

  • bh(Tree) >= h(Tree)/2。

    证明：每条路径上，除根节点外，至少有一半的节点时黑色。

    $N >= 2^{bh}-1 >= 2^{h/2}-1$，证明结束。

插入操作 #

需要尽可能避免改变红黑树的bh，避免重新平衡。因此新插入的节点颜色应为红色。但再次插入红节点时需要重新平衡，引出三种情况：

Case 1 #

新节点为红色，父节点为红色，父节点的兄弟节点也为红色。

解决方式：父代所有节点和祖父节点红黑颜色调换。

Case 2 #

新节点为红色且是父节点的右儿子，父节点为红色，父节点的兄弟节点为黑色。

解决方式：将新节点和其父节点进行一次旋转。

Case 3 #

新节点为红色且是父亲的左儿子，父节点为红色，父节点的兄弟节点为黑色。

解决方式：首先将父节点和祖父节点互换颜色，再将父节点作为新的根节点进行旋转。

AVL树 & 红黑树 #

  • step 1：向二叉搜索树中插入节点。
  • step 2：从底层到顶层，检查BF/颜色是否符合规定。

最坏情况相同，都需要进行两次rotation。

时间复杂度 #

$$ T=O(h)=O(logN) $$

删除操作 #

需要从底层向顶层平衡，AVL树的最坏情况是从叶子节点开始，直到整棵树平衡，消耗O(log N)；

而红黑树的优势在于，最坏情况只需要3次旋转即可。

二叉树的删除

红黑树的删除

对于degree 2 node，替换节点的同时保留原节点的颜色，不需要重新平衡。

对于degree 1 node，同样不需要重新平衡。

对于叶子节点，需要重新平衡，会出现一条路径黑色节点少一个的情况。因此需要向这条路径中加入一个黑色节点。

Case 1 #

删除的节点为黑色，兄弟结点为红色，父节点为黑色。

解决方法：先将兄弟节点与父节点颜色调换，再旋转兄弟节点，最后把新成为父节点的节点加上一个黑节点。（仍未解决）

Case 2 & Case 3 & Case 4 #

删除的节点为黑色，其兄弟节点均为黑色。

Case 2：兄弟节点的两个儿子均为黑色。

解决方法：首先将兄弟节点变为红色，面临两种情况。

①如果父节点为红色，将父节点变为黑色。

②如果父节点为黑色，更新x指向父节点，继续向x所在路径添加黑节点。这是最坏的情况，需要向上继续执行操作直到遇到case3或case4的情况或x最终为根节点，但不需要旋转，只进行了改变颜色的操作。

Case 3： 兄弟节点的左儿子为红色，右儿子为黑色。

解决方法：首先调换兄弟节点及其左儿子的颜色，再进行一次旋转。更新w指向旋转后的兄弟节点，可跳转到case4。

Case 4： 兄弟节点的右儿子是红色，左儿子任意。

解决方法：w节点的父亲和右儿子都变成黑色，w节点本身与原父亲节点的颜色相同，再进行一次旋转，最终解决问题。

共有3次旋转。

当存在大量插入或删除操作时，红黑树更好。
